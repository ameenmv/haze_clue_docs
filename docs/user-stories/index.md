# User Stories & Requirements

The `user_stories` repository defines the product scope, target audience needs, and agile sprint planning for the HazeClue project.

## Core Epics

### 1. Hardware Integration
- **As a user**, I want to seamlessly connect my smartwatch so that my sleep and HRV data are automatically synced.
- **As a user**, I want to connect my tDCS device so that I can initiate brain stimulation sessions directly from the app.

### 2. Cognitive Training
- **As a user**, I want to play memory and focus games so that I can actively train my brain.
- **As a user**, I want to listen to Binaural Beats so that I can enter a relaxed or focused state easily.

### 3. Data & Analytics
- **As a user**, I want to see a daily health tip based on my physiological data so that I can make informed lifestyle choices.
- **As a user**, I want to view my weekly progress on a graph so that I can track my cognitive improvements over time.

## Sprint Methodology
The team utilizes Agile methodologies, tracking progress through Jira/Trello boards, ensuring iterative delivery of features starting from core API and UI layouts, up to advanced AI integrations.
